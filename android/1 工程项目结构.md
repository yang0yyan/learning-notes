模块依附于项目，每个项目至少有一个模块，也能拥有多个模块。

编译运行app指的是运行某个模块，而非运行某个项目，因为模块才对应实际的app。

```tex
MyApplication/
├── .gradle/                   # Gradle构建系统缓存
├── .idea/                     # IntelliJ IDEA配置文件
├── app/
│   ├── build/                 # 构建输出文件
│   ├── libs/                  # 第三方jar库
│   ├── src/                   # 源代码目录
│   │   ├── androidTest/       # Android测试代码
│   │   ├── main/              # 应用的主要源代码
│   │   │   ├── java/          # Java源代码
│   │   │   ├── res/           # 资源文件（布局、字符串、图片等）
│   │   │   ├── AndroidManifest.xml  # 应用清单文件
│   │   │   └── ...            # 其他资源或源代码目录
│   │   └── test/              # 单元测试代码
│   ├── build.gradle           # 应用级别的构建脚本
│   └── ...
├── gradle/                    # Gradle wrapper相关文件
├── build.gradle               # 项目级别的构建脚本
├── gradle.properties          # 配置编译工程的命令行参数
├── gradlew                    # Unix系统的Gradle包装器启动脚本
├── gradlew.bat                # Windows系统的Gradle包装器启动脚本
├── local.properties           # 项目本地配置文件
├── settings.gradle            # 定义项目的设置
└── ...                        # 其他文件，如README, ProGuard配置等
```

## gradle

gradle是一个项目自动化构建工具，帮我们做依赖，打包，部署，发布，各种渠道的差异化管理等工作。
